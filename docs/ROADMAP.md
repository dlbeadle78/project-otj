# Development Roadmap

## Phase 1: Repository foundation

- Establish the GitHub repository as the master project record
- Add approved product documentation
- Add structured level and topic data
- Add the approved starter prompt library
- Import the current working source code

## Phase 2: Product consolidation

- Confirm one shared topic source for OTJ and related tools
- Review all generated prompt wording for accessibility and consistency
- Document prompt-building rules and safeguards
- Document the shared visual design system
- Record known defects and planned improvements as GitHub Issues

## Phase 3: Reliability and storage

- Introduce reliable cross-device storage for saved and generated prompts
- Define account, sign-in and data-retention requirements
- Test prompt creation, saving, copying, deletion and retrieval
- Add clear error and empty-state handling

## Phase 4: Release management

- Use version numbers for approved releases
- Test changes before they reach the main branch
- Keep release notes in `CHANGELOG.md`
- Maintain a recoverable stable version

## Future considerations

- EPA professional discussion question builder
- Shared navigation with the Virtual Teaching and Learning Hub
- Additional Learning Coach productivity tools
- Controlled exports and monthly prompt packs
